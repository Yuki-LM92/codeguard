About

CodeGuardとは？

Claude Code（AIコーディングアシスタント）を使っていると、
「このコマンドを実行してもいいですか？」という確認が頻繁に出てきます。

でも、コマンドの意味がわからなければ、判断できません。
CodeGuardはその瞬間に、コマンドの意味・影響・推奨アクションを
日本語でわかりやすく表示してくれるツールです。

⚡

リアルタイム解説

承認ダイアログが出た瞬間にブラウザに解説カードが表示されます

🇯🇵

完全日本語

コマンド名・オプションの意味・推奨アクションをすべて日本語で表示

🔍

手動でも調べられる

気になるコマンドを入力するだけで即座に解説が出ます

🌐

ブラウザで動く

インストール後は普通のブラウザで表示されます。特別な操作不要

How it works

どうやって動くの？

Claude Codeが記録するログファイルを監視し、コマンドを検出して解説するだけです。
AIへの操作には一切干渉しません。

Security

安全性について

「インストールして大丈夫？」という疑問にお答えします。

  • 🔒
    外部への通信はゼロです コマンドの解析はすべてあなたのPCの中だけで完結します。入力したコマンドや操作内容がインターネットに送られることは一切ありません。
  • 👁️
    Claude Codeの動作に干渉しません ログファイルを「読む」だけです。AIへの指示・コマンドの実行・ファイルの変更には一切関与しません。
  • 🏠
    ローカルホスト（127.0.0.1）のみで動作 あなたのPC以外からはアクセスできません。同じWiFiに繋がっている他の端末からも見えません。
  • 📦
    使用しているライブラリは3つのみ chokidar（ファイル監視）・express（サーバー）・ws（通信）の3つだけです。すべてオープンソースで広く使われている実績あるライブラリです。
  • 🛡️
    セキュリティ検査済み 依存パッケージの脆弱性ゼロ（npm audit確認済み）。セキュリティヘッダー・レート制限・入力長制限を実装しています。
Risk Levels

危険度の見方

CodeGuardはコマンドを5段階で評価します。色を見るだけで判断できます。

レベル 意味 推奨アクション 例
🟢 安全 許可して問題ありません ✅ 許可 ls, pwd, git status
🔵 ほぼ安全 通常の開発操作です ✅ 許可 npm install, mkdir, git add
🟡 注意 内容を確認してください ⚠️ 確認してから ファイル作成, git commit
🟠 警告 影響範囲を理解した上で判断 ⚠️ よく確認を rm（ファイル削除）, git push
🔴 危険 拒否を推奨します 🚫 基本は拒否 sudo, rm -rf ~/, chmod 777
Installation

導入手順

Node.js（v18以上）が必要です。Claude Codeを使っている方はすでにインストール済みです。

1
Node.jsのバージョンを確認

ターミナルを開いて以下を実行してください。v18以上が表示されればOKです。

node --version # v18.x.x 以上が表示されればOK
2
GitHubからダウンロード

以下のコマンドでCodeGuardをダウンロードします。

git clone https://github.com/Yuki-LM92/codeguard.git cd codeguard
3
インストール

1行のコマンドでインストール完了です。

npm install -g .
4
起動する

これだけです。ブラウザが自動で開きます。

codeguard

Uninstall

削除したいときは

1行のコマンドで完全に削除できます。PCへの影響は残りません。
